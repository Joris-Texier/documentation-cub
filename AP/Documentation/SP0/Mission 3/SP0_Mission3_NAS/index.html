    SP0 – Mission 3 : Mise en place d’un serveur NAS (NASEPOKA)¶

    Auteur : Joris Texier
    Contexte : EPOKA
    Version : 1
    Date de rédaction : 15 septembre 2025

    Sommaire¶

    1. Création de la VM sur Nutanix
    2. Installation de TrueNAS
    3. Configuration Réseau de TrueNAS
    4. Création d’un Pool dans TrueNAS
    5. Intégration de TrueNAS dans le domaine Active Directory
    6. Test et Configuration SMB

    Création de la VM sur Nutanix¶

    1. Cliquer sur “Create VM from catalog item”.

    1. Choisir l’image ISO de TrueNAS.
    2. Configurer les paramètres comme suit :
    3. Nom : NASEPOKA
    4. RAM : 4 Go
    5. CPU : 2 vCPU

    • Disque : 20 Go
    • Réseau : Projet F

    1. Ajouter 3 disques supplémentaires de 20 Go chacun.

    2. Définir le fuseau horaire sur Europe/Paris.

    Installation de TrueNAS¶

    1. Lancer la VM.

    1. Appuyer sur Entrée ou attendre la fin du compte à rebours.

    2. Sélectionner Install/Upgrade.

    1. Choisir le disque NUTANIX VDISK – 20.0 GiB.

    1. Sélectionner Yes.

    1. Définir le mot de passe administrateur (root).

    1. Sélectionner les options par défaut jusqu’à la fin de l’installation.

    1. Cliquer sur Shutdown System.

    1. Retirer l’image ISO avant de redémarrer.

    Configuration Réseau de TrueNAS¶

    1. Attribuer une adresse IP fixe :
    2. IP : 172.16.56.20/24
    3. Passerelle : 172.16.56.254
    4. Vérifier la connectivité via un navigateur :
      http://172.16.56.20

    1. Depuis l’interface Web, aller dans Network → Interfaces → Edit.

    1. Définir la description, l’adresse IP et valider avec Apply.

    1. L’adresse IP est désormais fixe.

    Création d’un Pool dans TrueNAS¶

    1. Aller dans Storage → Pools → Add.

    1. Cliquer sur Create new pool puis Create Pool.
    2. Sélectionner les disques à inclure dans le RAID.
    3. Choisir le type de RAID selon le nombre de disques :
    4. Stripe (RAID0) : aucune tolérance de panne.
    5. Mirror (RAID1) : perte d’un disque tolérée.
    6. RAIDZ1 (RAID5) : perte d’un disque tolérée.

    • RAIDZ2 / RAIDZ3 : tolérance plus élevée.
    • Nommer le pool et cliquer sur Create.

    Intégration de TrueNAS dans le domaine Active Directory¶

    Configuration du DNS sur TrueNAS¶

    1. Aller dans Network → Global Configuration.
    2. Saisir :
    3. Serveur DNS : 172.16.56.1
    4. Domaine : local.epoka6.lan

    Configurer l’Active Directory¶

    1. Aller dans Directory Services → Active Directory.

    1. Remplir :
    2. Domain Name : local.epoka6.lan
    3. Domain Account Name : Administrateur
    4. Password : Etudiant_007
    5. Computer Name : TRUENAS
    6. Cocher Activer.

    Joindre le domaine¶

    • Cliquer sur Save.
    • TrueNAS rejoint automatiquement le domaine AD.

    Vérification de l’intégration¶

    1. Ouvrir le Shell dans TrueNAS.
    2. Vérifier la connectivité : 
      ping 172.16.56.1
wbinfo -u
wbinfo -g

    1. Sur l’Active Directory, vérifier la présence du serveur TRUENAS.

    Test et Configuration SMB¶

    Création du Dataset¶

    1. Aller dans Storage → Volumes → Add Dataset.
    2. Nommer le dataset : test.
    3. Définir les autorisations :
    4. Type : Groupe
    5. Nom : LOCAL\admins du domaine
    6. Droits : Lecture, Écriture, Modification
    7. Cocher Appliquer récursivement puis Enregistrer.
    8. Vérifier avec : 
      getfacl /mnt/Pool/test

    Configuration du Partage SMB¶

    1. Aller dans Sharing → Windows (SMB) Shares → Add.
    2. Chemin : /mnt/Pool/test
    3. Nom : Test
    4. Cochez Activé et Utiliser les ACL.
    5. Enregistrer.

    Vérification de l’Accès depuis Windows¶

    1. Sur un poste du domaine, ouvrir l’Explorateur et entrer : 

      \\172.16.56.20

    2. Se connecter avec : 

      elise.arbourg@local.epoka6.lan
Mot de passe : etudiant_007

    3. Tester la création, modification et suppression de fichiers.
